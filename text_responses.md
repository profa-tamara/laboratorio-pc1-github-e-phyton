## Questão 1

a)Porque ele utiliza um software (intérprete) que lê o cadigo linha por linha, e não todo de uma vez, durante a execução do programa. Ou seja, não há a necessidade de um compilador, visto que o programa é lido linha por linha.

c)Porque é uma linguagem que da suporta a programação orientada à objetos, que é um paradigma desenvolvido como forma de tratar o rápido aumento no tamanho e complexidade dos sistemas de software e facilitar a modificação desses sistemas grandes e complexos ao longo do tempo.

d)Porque o próprio interpretador do Python determina o tipo de dados que uma variável recebe, sem a necessidade que o usuário da linguagem insira que tipo determinada variável é. ## Questão 2

Markdown é um formato de simples de markup (marcação de texto). A ideia é marcar um texto informando o que é importante, o que é um tópico, o que são links e imagens, sem a necessidade de utilizar marcações mais complexas, como o HTML. É um formato bastante trivial e apesar de criado por pessoas que trabalham com programação, o markdown hoje aparece de inúmeras formas e em vários lugares.

## Questão 3

Um repositório contém todos os arquivos do seu projeto e o histórico de revisão de cada arquivo, onde se pode discutir e gerenciar o trabalho do projeto dentro do repositório. Você pode possuir repositórios individualmente ou compartilhar a propriedade de repositórios com outras pessoas em uma organização. É possível restringir quem tem acesso a um repositório escolhendo a visibilidade do repositório.

Branch são ramificações usada para isolar o trabalho de desenvolvimento sem afetar outros branches no repositório. Os branches permitem que você desenvolva recursos, corrija erros ou experimente com segurança novas ideias em uma área contida do seu repositório, sem afetar o código principal.
